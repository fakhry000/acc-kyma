# Operations notes

## Public URLs

Every instance must keep these aligned to the external HTTPS hostname:

- `N8N_HOST`
- `N8N_PROTOCOL=https`
- `N8N_PORT=5678`
- `N8N_EDITOR_BASE_URL`
- `WEBHOOK_URL`
- `webhook.url`

This repo keeps those values in the instance values file and the core secret so links generated by n8n do not point to internal service names or `:5678` URLs.

## Redis isolation

Both instances share one Redis server, but each instance must keep its own prefix:

- `n8n-hr`
- `n8n-fi`

## PostgreSQL isolation

Both instances share one PostgreSQL instance and one database, but each has its own schema and DB user.

## Argo CD login on Kyma

- set `server.insecure=true` before exposing Argo CD
- use `argocd admin initial-password -n argocd` for the first login


## Chart notes

The official `n8n-io/n8n-hosting` chart in this repo is pinned and uses `database.type: postgresdb`, matching the validated chart schema from your previous working setup.


## Current ownership model

PostgreSQL and Redis are pre-provisioned manually and are intentionally outside Argo CD management in this cleaned repository.
